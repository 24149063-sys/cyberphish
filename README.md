# CyberPhish

A Flask-based web app for detecting phishing URLs using a machine-learning
classifier, with login, a dashboard, single/bulk URL scanning, and downloadable
reports.

## Setup

1. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

2. **Train the model** (creates `model.pkl` and `vectorizer.pkl` in the project root)
   ```bash
   python models/train_model.py
   ```
   This trains a Logistic Regression classifier on character n-gram TF-IDF
   features of URLs, using a built-in synthetic phishing/legitimate dataset.
   Swap in a real dataset (e.g. PhishTank + a top-sites list) by editing
   `load_dataset()` in `models/train_model.py` for production use.

3. **Run the app**
   ```bash
   python app.py
   ```
   The SQLite database (`database/phishing.db`) and its tables are created
   automatically on first run, from the schema in `database/phishing.sql`.
   A default account is seeded:
   - **Username:** `admin`
   - **Password:** `admin123`

   Visit `http://127.0.0.1:5000`.

## Features

- **Login** вЂ” session-based auth, passwords hashed with `werkzeug.security`.
- **Dashboard** вЂ” total scans, phishing vs. legitimate counts, recent activity.
- **Scan / Upload** вЂ” check a single URL, or upload a CSV (one URL per row)
  for bulk scanning.
- **Reports** вЂ” browse past scans (filterable by upload batch) and export
  results to CSV.

## Notes for production use

- Replace the synthetic training data with a real, regularly-updated dataset.
- Set `CYBERPHISH_SECRET_KEY` as an environment variable instead of using the
  default dev key.
- Swap SQLite for MySQL/PostgreSQL if you need concurrent multi-user access
  (schema in `database/phishing.sql` is portable with minor type tweaks).
- Put the app behind a production WSGI server (gunicorn/uWSGI) instead of
  Flask's built-in dev server.
